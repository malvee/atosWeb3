Research - Topics searched and found

    APIs

    APIs for various social platforms were identified in the first sprint. They all support HTTP REST protocol so that any programming language can make and retrieve requests.

      Twitter

      The Twitter micro-blogging service provides two RESTful APIs. The Twitter REST API methods allow developers to access core Twitter data. This includes update timelines, status data and user information. The search API methods allow developers to interact with Twitter search and trends data. The APIs currently support following data formats: XML, JSON, RSS and the Atom syndication formats. The four main objects that are encountered in the API are Tweets, Users, Entities and Places. Resources were identified for various programming languages:
      • Java - hbc library, temboo library
      • C++ - twirlcurl
      • JavaScript/Node.js - twitterJSclient library

      Facebook

      The Facebook API is a platform for building applications available to the members of this social network. The API allows applications to become more user-centered by publishing activities to news feed and user profile by using social connections and profile information (SUBJECT TO USER PRIVACY SETTINGS). Users can add social context to applications by utilizing profile, friends, page, group, photo and event data. The API uses RESTful protocol and responses are localized in XML format.

      Youtube

      The problem with Youtube is it does not have hashtag searching so searches cannot be made on comments or video description. Only the basic keyword youtube search can be done which returns a list of videos with best match. Youtube has two relavant APIs : Data API and Analytics API. The YouTube Data API (v3) lets incorporate YouTube functionality into own application. The API can be used to fetch search results and to retrieve, insert, update, and delete resources like videos or playlists. There is a "search.list" method in Data API but it only allows searching through keywords and returns a list of videos. The analytics API allows viewing statistics, popularity metrics, and demographic information for YouTube videos and channels but this is not quite relevant to the project.

      Instagram

      - The Instagram API allows searching for photos by hashtags, supports REST request through api.instagram.com endpoint and JSON response and uses OAuth 2.0 protocol for authentication.

    Test Data

    Django

    Django is a high level Python Web framework. It helps in building high-performing, elegant Web applications quickly because of focus on automation and DRY (Don't Repeat Yourself) principle. Some interesting Django features are:
    • Object-relational Mapper - Data models are defined completely in Python and there is also a rich, dynamic-database access API.
    • Elegant URL design - Django provides neatly designed URLs with no framework-specific limitations.
    • Template system - Django has a powerful, extensible and design-friendly template language to separate design, content and Python code. Data models are defined completely in Python and there is also a rich, dynamic-database access API.
    • Cache system - Performance is increased by hooking into memcached or other cache frameworks.
    • Internationalization -Django supports multi-language applications by allowing specification of translation strings and linking language-specific functionalities.
    • Sites like Instagram, Mozilla, OpenStack, Pinterest, etc. have had tremendous success with Django and the team expects the same with this project as well.

    Sentiment Analysis

    The team looked at various APIs like BigML, Google Prediction API, ViralHeat etc that could be used to perform sentiment analysis on tweets. Some of the APIs were not open-source and some were complicated to the extent that limited the team from achieveing protoype completion. A perfect balance was spotted in DatumBox. It is open-souce and has a dedicated twitter sentiment analysis API. The API is very simple and extremely powerful to implement.

    PHP

    The DatumBox API is supported to work with PHP and since the team is well known with back-end PHP development, the prototype was finished comfortably well within time. This meant that there was no need to learn a brand new or over-complicated technology stack. Hence the choice of PHP for handling all back-end tasks and to display dynamic data for the application was appropriate. The team also looked at PHP web frameworks such as Laravel but at such a late phase of prototype development Laravel would not have made it easier or added any special features. So the choice of working with simple and effective PHP coding without the use of any frameworks was also appropriate.
